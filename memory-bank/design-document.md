# 项目文档

## 一、项目名称

应聘信息追踪系统（Application Tracker）

---

## 二、项目背景与目标

在寻找工作的过程中，个人往往需要同时维护多条投递记录，包括不同公司、不同岗位、不同流程阶段。使用传统表格工具难以支持灵活筛选、状态追踪和后续扩展。

本项目旨在构建一个 **面向个人的投递流程管理工具**，用于集中记录、追踪和管理所有重要的应聘投递信息，提高信息可见性与决策效率。

---

## 三、功能说明

### 1. 页面结构

本项目包含两个页面：

- **投递记录表页面（核心页面）**
  - 以卡片 / 表格形式展示所有投递记录
  - 支持新增、编辑、删除投递信息
- **日程表页面（预留）**
  - 用于后续展示笔试、面试等时间节点
  - 当前阶段仅保留页面入口，不实现具体功能

---

### 2. 投递记录表功能

#### （1）卡片展示内容

投递记录表页面以「单条投递卡片」作为最小信息单元，每一张卡片对应一次独立的投递行为，必须完整展示以下信息字段：

- 公司名称（companyName）：作为卡片的主标题展示。
- 岗位名称（jobTitle）：如：前端开发。
- 工作城市（city），公司级别（companyLevel），直接以文本形式展示，如：大厂 / 中厂 / 小厂 / 国企 / 外企。
- 投递渠道（applyChannel），如：官网 / 内推 / 邮箱。
- 投递时间（applyDate），用于排序，同时在卡片中以可读日期形式展示。
- 投递链接：点击链接（applyLink）可以跳转到招聘页面
- 当前流程（processStatus），如：投递中 / 一面结束 / 二面中。
- 信息记录（interviewRecord）：弱结构文本记录，不拆分为多字段，例如：一面 3.1，二面 3.3。
- 投递结果（result）：可能值为 offer / 一面挂 / null。当 result 为 null 时，表示该投递仍处于流程中。

---

#### （2）交互行为

点击公司名称
跳转至对应的招聘网站（applyLink）。
单击卡片主体
进入编辑状态，可修改该条投递的全部字段。
右键卡片
弹出删除操作菜单，点击删除后移除该条投递记录。

---

### 3. 筛选、分类与排序

- **分类（公司级别）**
  - 大厂 / 中厂 / 小厂 / 国企 / 外企
- **筛选条件**
  - 根据投递结果是否为空（result = null）筛选"流程中"的投递
- **排序规则**
  - 按投递时间（applyDate）排序

---

## 四、数据库数据结构设计

本项目使用 **IndexedDB** 作为本地数据持久化方案，采用 **单对象仓库（Object Store）结构** 存储所有投递记录。
仓库名称：`applications`

### 字段设计

| 字段名          | 含义             | 存储类型           | 说明                             |
| --------------- | ---------------- | ------------------ | -------------------------------- |
| `id`            | 投递记录唯一标识 | string / uuid      | 数据库自动生成                   |
| `companyName`   | 公司名称         | string             | 如：字节跳动                     |
| `jobName`       | 岗位名称         | string             | 前端开发                         |
| `city`          | 工作城市         | string             | 北京 / 上海                      |
| `companyLevel`  | 公司级别         | string             | 大厂 / 中厂 / 小厂 / 国企 / 外企 |
| `applyChannel`  | 投递渠道         | string             | 官网 / 内推 / 邮箱               |
| `applyLink`     | 招聘链接         | string             | 点击可跳转                       |
| `applyDate`     | 投递时间         | string（ISO Date） | 用于排序                         |
| `processStatus` | 当前流程状态     | string             | 如：投递中 / 一面结束            |
| `Record`        | 信息记录         | string             | 纯文本，如：一面 3.1，二面 3.3   |
| `result`        | 投递结果         | string 或 null     | offer / 挂 / null                |
| `remark`        | 备注             | string             | 可选补充说明                     |
| `createdAt`     | 创建时间         | string（ISO Date） | 系统生成                         |
| `updatedAt`     | 更新时间         | string（ISO Date） | 最近修改时间                     |

### 设计说明

- 一条记录对应一次独立投递行为
- 允许公司名称与招聘链接重复
- 面试时间采用弱结构文本记录，避免过度建模
- `result = null` 作为"流程中"投递的核心判断条件

### IndexedDB 说明

- 使用 `idb` 或 `Dexie.js` 库简化 IndexedDB 操作
- 数据完全存储在用户浏览器本地，无需网络连接
- `id` 字段使用 `keyPath` 自动生成（建议使用 UUID 或自增主键）
- 支持按 `applyDate`、`companyLevel`、`result` 等字段建立索引，优化查询性能
- 字段设计直接服务于卡片展示、筛选与编辑功能
